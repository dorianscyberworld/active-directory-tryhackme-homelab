# Lab 002 – Users and Groups

## Objective
Understand how users and groups are managed inside Active Directory.

---

## What I Learned
Active Directory uses users and groups to manage access to systems, resources, and permissions efficiently.

Instead of assigning permissions to each user individually, administrators assign permissions through group membership.

---

## Key Concepts
- User Accounts
- Security Groups
- Group Membership
- Access Permissions
- Role-Based Access

---

## Why This Matters
Managing users and groups properly improves security and simplifies administration.

Help desk teams commonly work with:
- Account creation
- Password resets
- Account unlocks
- Group membership changes
- Access provisioning

Cybersecurity teams focus on:
- Privileged groups
- Access reviews
- Unauthorized access detection
- Least privilege enforcement
